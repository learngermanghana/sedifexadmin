Integration delivery sequence (high confidence)

Recommended order for near-term product and operational payoff as of April 4, 2026.

  1. Ship real integration API keys + revoke/rotate UI
    • Store-scoped key generation (prefix + hashed secret at rest).
    • In-app metadata: key name, created time, last used time, status.
    • One-time secret reveal at creation.
    • Immediate revoke and documented rotate flow with overlap window.
    • Audit log entries for create/revoke/rotate.
  2. Ship WordPress plugin MVP
    • Plugin settings for API key + store selection.
    • Product shortcode/block rendering.
    • Sync health panel (last success/failure, item count, next retry).
    • 30–120 second cache window + manual sync trigger.
  3. Ship basic product webhooks
    • product.created, product.updated, product.deleted.
    • Per-endpoint signing secret with X-Sedifex-Signature.
    • Retry policy with exponential backoff.
    • Delivery log view with attempts and response codes.
    • Copy/paste signature verification docs.

Suggested cadence

  • Milestone A (Weeks 1–2): API keys + revoke/rotate UI + audit logs.
  • Milestone B (Weeks 3–4): WordPress MVP + sync health + onboarding guide.
  • Milestone C (Weeks 5–6): Webhooks + signing + delivery logs + verification docs.

Dependency note: if overlap is required, keep key-management primitives (creation, hashing, revoke, rotate, audit) as non-negotiable before broad partner rollout.

WordPress install guide (Sedifex Sync)

Use this setup to connect a WordPress site to a Sedifex product catalog. If you are using Next.js on Vercel, use quickstart.html instead.

What this setup does

  1. Uses a Sedifex integration API key.
  2. Calls the integrationProducts endpoint.
  3. Renders products in WordPress via shortcode.
  4. Applies dedupe + fallback + cache strategy.

Prerequisites

  • A WordPress site where you can install plugins.
  • A Sedifex integration API key for the target store.
  • Sedifex API base URL for your deployed Functions endpoint.

Install steps

  1. Install plugin scaffold. You can use Code Snippets or your preferred plugin workflow. A baseline is included in wordpress-plugin-sedifex-sync.php.
  2. Add a frontend/client script that requests integrationProducts?storeId=<storeId> with Authorization: Bearer <integration_key>.
  3. Create and place a shortcode like [sedifex_products].
  4. Store config values: SEDIFEX_API_BASE_URL, SEDIFEX_STORE_ID, and SEDIFEX_INTEGRATION_KEY.
  5. Validate rendering, out-of-stock handling (stockCount <= 0), and fallback behavior under fetch failure.

Recommended hardening

  • Add a visible “Last synced at” timestamp.
  • Alert on repeated sync failures.
  • Rotate integration credentials periodically via Account → Integrations.

Team and access model

  • Invite developer collaborators per workspace/store assignment.
  • Limit key management and webhook admin access to least privilege roles.
  • Audit invite, role, key, and webhook changes during release milestones.
